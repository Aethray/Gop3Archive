# 🎈 EasyClick 德州扑克自动脚本 (炸弹气球模式)

[![Version](https://img.shields.io/badge/version-v1.0.2-blue.svg)](#v102-终极策略与ui重构)
[![EasyClick](https://img.shields.io/badge/EasyClick-ES5%20%7C%20Rhino-orange.svg)](https://ieasyclick.com/)

本项目是一套基于 EasyClick (Rhino 引擎) 开发的高级自动化德州扑克（炸弹气球模式）挂机脚本。项目实现了从底层 UI 控件交互、基于 GTO 的数学算牌决策模型，到高级悬浮窗提醒及代码级热更新的全栈自动化功能。

### 🌟 v1.0.2 终极策略与UI重构
*更新日期：2026-05-08*

**核心代码变更与功能新增：**
- **高颜值自定义 Toast (穿透版)**: 彻底重写了 `tm.toast`。新增悬浮窗级别的消息提示，支持动态防抖热更新、自定义圆角/颜色，并完美支持 **事件穿透 (`passThrough: true`)**，不再遮挡游戏操作。
- **UI 布局重构 (WebView 底部遮挡修复)**: 放弃纯 HTML 渲染，将其重构为纯正的 Android 原生 `ScrollView + TableLayout` 的 XML 布局，并增加 `paddingBottom="100dp"`，彻底解决 720p 手机端底部导航栏遮挡文档的问题。
- **全周期打牌策略内嵌**: 增加极其详尽的翻前(Pre-Flop)至河牌(River)的打牌策略与分数换算矩阵文档。
- **静默热更新机制**: 取消了打扰用户的弹窗更新，改为后台子线程静默下载，并在下载完成后安全重启脚本。
- **结算与领奖优化**: 牌局结束自动检测并领取团队点数奖励，动态计算下一次领取时间。

**建议操作：**
> 推荐所有用户升级至此版本，享受无感热更新及最流畅的 UI 交互体验。

### 🚀 v1.0.1 性能优化与引擎编译版
*更新日期：2026-05-05*

**核心代码变更与功能新增：**
- **单选框联动下拉框修复**: 修复了 EasyClick 在弹窗上下文中 `findViewWithTag` 寻找不到子视图的 BUG。将上下文正确指向 `page.home`，实现了点击“钥匙”单选框自动将下拉框切换为“250M”的丝滑联动。
- **ES5 Dex 编译支持**: 针对底层的 `daily` 对象及业务函数进行了严格的 ES5 降级规范，摒弃所有 `let/const`，确保项目可以 100% 成功编译成 `.dex` 字节码文件，极大提升脚本运行效率与防破解能力。
- **日志节流防御 (Throttling)**: 在 `Tm` 工具类中新增 `logThrottle` 方法，彻底解决了高频 `while(true)` 循环找色导致的控制台刷屏、CPU 满载发热问题。
- **前台应用精准检测**: 修复了安卓底层 `dumpsys activity top` 的误判 BUG，改用精准抓取屏幕焦点的逻辑，防止脚本在后台错误运行。

**建议操作：**
> 适合需要长时间(24小时+)挂机的用户，内存与 CPU 控制极佳。

### 🚩 v1.0.0 核心状态机稳健版
*更新日期：2026-05-01*

**核心代码变更与功能新增：**
- **基础状态机构建**: 确立了 `home`(大厅)、`room`(选场)、`load`(加载)、`cards`(打牌) 的无缝流转机制。

- **全局看门狗与防卡死**: 引入了三分钟全局超时监控机制 (`watchdogCheck`)，一旦发现游戏画面卡死或断网，自动清理内存并重启 App。

## 🛠️ 项目架构与技术栈
- **核心引擎**: JavaScript (严格遵循 ES5 标准，兼容 Rhino 引擎)
- **UI 层**: Android 原生 XML 动态生成 + 纯代码构建 (View & WindowManager)
- **视觉识别**: EasyClick 多点找色 (`findMultiColor`) + OCR 文字识别
- **架构设计**: 面向对象编程 (OOP) 原型链挂载、全局单例状态机
